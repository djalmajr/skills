# /wiki-init para o reserva-pwa

## Contexto

Após o `/agile-intake` ([2026-05-12-intake-reserva](./2026-05-12-intake-reserva.md)), inicializar a wiki **antes** de seguir para `/agile-roadmap` — várias decisões do roadmap dependem de regras de domínio (capacidade, no-show, janela de reserva) que devem viver na wiki, não no código nem no roadmap.

## O que tentei

```
/wiki-init "Inicializar wiki para /Users/djalmajr/Developer/zommehq/reserva-pwa/.
 PWA de reserva de mesa estilo OpenTable. Domínio: capacidade por turno,
 no-show, janela de reserva, blackout dates, cancelamento.
 Projeto cobaia para validar fluxo das skills agile + wiki."
```

## O que aconteceu

1. **Doctor dry-run** (`bun skills/wiki-init/scripts/wiki-init.ts doctor --project ...`) — reportou:
   - `wiki_path: wiki (missing)` — esperado, projeto novo
   - `qmd_index: reserva-pwa` — inferido do basename, exatamente o default que eu escolheria
   - `recommended_topology: local-wiki (suggested)` — correto para projeto cobaia
   - 3 harnesses detectados (claude, codex, opencode)
   - 22 arquivos planejados (configs + hooks para cada harness + MCP + wrapper QMD + manifest)
   - QMD versão 2.1.0 já instalado e patched OK
2. **Decisões registradas em voz alta** (usuário em modo "não pausar para clarificar"): local-wiki, index `reserva-pwa`, 3 harnesses.
3. **Install --write** — criou os 22 arquivos sem erro. Tudo `present` no doctor seguinte.
4. **Wiki content** — `wiki_path: wiki (exists)` ainda continuava `(missing)` após o install. Por quê? Porque a `wiki-init` configura **infraestrutura**, não o **conteúdo** do wiki. Tive que criar manualmente `wiki/index.md`, `wiki/CONVENTIONS.md`, `wiki/log.md`, subpastas por audiência (`business/`, `apps/`, `ops/`, `data/`, `sources/`) e `raw/index.md`. Estrutura inferida da leitura do `wiki-ingest/SKILL.md` (que assume essa convenção).
5. **QMD index init** (passo 9 do SKILL.md): `<wrapper> collection add wiki --name reserva-pwa --mask "**/*.md"` → 3 docs indexados. `<wrapper> update` → ok. `<wrapper> embed` → 4 chunks de 3 documents embedded em 2s.
6. **Doctor final** — tudo verde: `wiki_path (exists)`, todos os files `present`, `installed drift: none`, QMD `status: ok`.
7. Adicionei contexto não-managed ao `CLAUDE.md` do projeto (referência ao intake, ao journal, e domínio a ser populado). Commit feito.

## O que funcionou

- **Doctor é excelente como entrada.** Read-only, exaustivo, lista planned actions explicitamente. Confiança alta de que `--write` não vai surpreender.
- **Topologia + index default vieram corretos.** O script já sugeriu local-wiki e `reserva-pwa` como index — o usuário só precisa confirmar. Em modo "não pause", a confirmação é decisão minha, mas os defaults estão certos.
- **Install é idempotente.** Re-rodar `doctor` depois do install mostrou `installed drift: none detected` — sinal claro de sucesso.
- **QMD wrapper funcionou first-try.** Comando `collection add` + `update` + `embed` na sequência prescrita pelo SKILL.md, sem fricção.
- **Patches de QMD detectados.** O doctor reportou `patch embed model: ok`, `patch MCP --index: ok`, `patch hyphenated vec/hyde: ok` — sinaliza que a patch suite está saudável.

## O que travou ou ficou ambíguo

- **`[[finding-candidate]]` — A skill configura infraestrutura mas não semeia o conteúdo do wiki.** `wiki_path: wiki (missing)` continuou após `--write`. O SKILL.md menciona "If the target project needs an index, initialize QMD..." mas não diz "crie índice/conventions/log mínimos". Tive que inferir a estrutura mínima de `wiki-ingest/SKILL.md`. Hipótese: a `wiki-init` poderia ter um passo "seed wiki" opcional, ou pelo menos um *checklist explícito* listando os arquivos mínimos esperados (`index.md`, `CONVENTIONS.md`, `log.md`, subpastas por audiência) e onde o template está. **Este é o segundo `[[finding-candidate]]` sobre lacuna entre o que a skill faz e o que o usuário precisa fazer manualmente depois.**

- **`[[finding-candidate]]` — O repo `skills/` (este meta-repo) não dogfooda seu próprio padrão.** O `CLAUDE.md` do `skills/` referencia `wiki/index.md`, `wiki/CONVENTIONS.md`, `wiki/log.md` — mas o diretório `wiki/` **não existe** neste repo. Isso é uma divergência meta-curiosa: o repo das skills wiki não usa wiki. Pode ser intencional (esse é um repo de skills, não um repo de conhecimento de domínio), mas o `CLAUDE.md` referencia como se existisse. Hipótese: ou remover a referência (se intencional) ou rodar `/wiki-init` neste repo para coerência (dogfooding). Vale promover a finding sobre isso porque afeta credibilidade do padrão.

- **`[[finding-candidate]]` — Repetição da observação anterior do intake: path relativo + CWD.** O wrapper QMD foi criado em `/Users/djalmajr/.local/share/skills/qmd/wrappers/reserva-pwa-qmd` (path absoluto do skill cache do usuário). O comando `<wrapper> collection add <wiki-path>` aceita path absoluto, mas o SKILL.md não enfatiza isso — passei `/Users/djalmajr/Developer/zommehq/reserva-pwa/wiki` para evitar ambiguidade. Hipótese: documentar nos exemplos do SKILL.md que paths absolutos são a forma segura. Já é o **segundo journal** apontando para o mesmo padrão (path/CWD ambiguity) — pronto para virar finding consolidado.

- **`[[finding-candidate]]` — Spinner do `qmd embed` deixa lixo na saída.** A saída inclui sequências de escape ANSI (`[?25l[?25h██████████████████████████████ 100%`) que poluem logs quando capturados. Pequeno, mas se uma sessão futura quiser parsear saída programaticamente vai sofrer.

# Skill Feedback: `agile-epic` — three refinements

**Observed during:** project `reserva-pwa`, epic decomposition session 2026-05-12 (Foundation initiative)
**Date:** 2026-05-12
**Reporter:** agent (Claude Opus 4.7, 1M context)

## Context

- **Workflow being executed:** decomposing the Foundation initiative into 5 stories.
- **Artifact or code involved:** `reserva-pwa/planning/mvp-reserva/epics/01-foundation/{00-overview.md, 01-...05-...md}`.
- **Skill/template affected:** `skills/agile-epic/SKILL.md` + `templates/epic.md`.

## Observation

- **Expected behavior:** turn an initiative into an epic overview + N story files; produce a roadmap diagram + risks + acceptance criteria; recommend `/agile-story` for the first story.
- **Actual behavior:** core flow worked well — 6 files produced cleanly. Three secondary issues surfaced.
- **Evidence (3 sub-findings):**
  1. **User-passed save path vs. skill-convention path.** User said "save under `planning/mvp-reserva/foundation/`"; SKILL.md prescribes `planning/<initiative>/epics/NN-<epic-name>/`. The agent silently followed the convention; the user's literal path was overridden without surfacing. Source: [2026-05-12-epic-foundation](../journal/2026-05-12-epic-foundation.md).
  2. **Mermaid `gantt` template assumes scheduled work.** Template suggests `gantt` with `dateFormat YYYY-MM-DD`. For a solo dev with no calendar deadline, gantt is fake precision. The agent substituted a `flowchart` showing dependencies. Same source.
  3. **`## Files (exact paths, action, reason)` invites false precision.** At epic-time, listing 10–17 exact file paths per story implies a level of certainty the agent does not have. Some paths inferred (e.g., generated `routeTree.gen.ts`) may end up named differently in execution. Source: [2026-05-12-epic-foundation](../journal/2026-05-12-epic-foundation.md) + [2026-05-12-story-bootstrap](../journal/2026-05-12-story-bootstrap.md) (both observed it).
- **Impact:** path conflict can lead to silent file misplacement (artifacts where the user did not expect); gantt produces dead diagrams in solo-dev contexts; over-specified `Files` invites scope-creep validation during execution.

## Diagnosis

- **Root cause:** SKILL.md is opinionated about save path and Mermaid type but does not surface either as a *decision point* with the user. Template's `Files` section is uniformly strict; lacks a "core vs. probable" distinction.
- **Scope of issue:** every epic decomposition.
- **Repeated or one-off?** Single epic so far, but all three are structural — they will recur on the next epic.

## Recommended action

- [x] Refine existing skill/template
- [ ] Merge / Split / Deprecate / Remove / Create

## Proposal

- **Target files:** `skills/agile-epic/SKILL.md`, `skills/agile-epic/templates/epic.md`.
- **Proposed change:**
  1. **Path-conflict gate (SKILL.md).** In the "Where to save" section, add: "If the user passes a save path that does not match the convention `planning/<initiative>/epics/NN-<epic-name>/`, surface the conflict before writing. Honor the user's path only after explicit confirmation; otherwise apply the convention and tell the user why."
  2. **Mermaid variant guidance (SKILL.md + template).** Update "Define roadmap" to: "Use `flowchart` for dependency-driven roadmaps without dates; use `gantt` only when team scheduling is real. Solo dev → almost always `flowchart`." Provide both as alternative blocks in the template.
  3. **Refine the `Files` section (story files).** Split into two sub-lists or annotate paths: `core` (handcrafted, agent commits to the path) and `probable` (inferred or tool-generated; exact name may shift). Add one line of guidance: "If a path is generated by tooling or has a non-deterministic name, mark it as `probable` and resolve at execution time."
- **Risk/tradeoff:**
  - Low. Documentation + template tweaks; no behavior change.
  - Slight noise risk in `Files` if the core/probable distinction is over-applied — keep guidance terse.
- **Alternatives considered:**
  1. Always honor the user's path silently. Rejected — surfaced inconsistency is the point of the convention.
  2. Drop the Mermaid gantt entirely. Rejected — gantt is still right for team work with deadlines.
  3. Make `Files` aspirational only. Rejected — exact paths at epic-time catch scope creep too, when they hold.

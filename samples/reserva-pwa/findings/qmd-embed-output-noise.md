# `qmd embed` output includes ANSI sequences when captured

## Evidence

- [2026-05-12-wiki-init](../journal/2026-05-12-wiki-init.md) — captured output contains `[?25l[?25h██████████████████████████████ 100%` (ANSI escape sequences for hide/show cursor + progress bar). Pollutes logs when stdout is redirected to a file or parsed.

## Pattern observed

The `qmd embed` binary emits a spinner/progress bar with ANSI escape sequences without checking whether stdout is a TTY. Common pattern in immature CLIs — these should be suppressed when `!isatty(stdout)`.

## Why it matters

- Low priority. Does not block usage.
- Causes friction in automation (CI scripts, hooks, output parsing).
- Likely symptom: probably affects `qmd update` and other progress-bearing commands too.

## Refinement hypothesis

**Upstream.** This is binary behavior in `qmd`, not in the skills here. Correct path:

1. Open an issue at `https://github.com/tobi/qmd.git` with the repro.
2. Local workaround: filter ANSI when running in non-interactive mode (e.g., `qmd embed 2>&1 | sed 's/\x1b\[[0-9;?]*[a-zA-Z]//g'`). Could be wrapped into the wrapper generated by `wiki-init`.

## Expected validation

- An upstream issue with a minimum reproducer is open.
- If the upstream patch is slow, consider adding an ANSI filter inside the generated wrapper.
